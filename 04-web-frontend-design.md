# Web 端前端设计方案 v0.1

本文档记录 `note-remeber` Web 端第一版的前端页面、布局、交互、组件、状态和响应式设计。

本文档基于：

```text
01-web-product-spec.md
02-web-technical-environment.md
03-web-backend-design.md
```

当前目标是先把 Web 端做成可用的阅读研究工作台，后续 App 端可以复用核心交互和数据模型。

## 1. 前端总体定位

Web 端不是营销网站，也不是展示页。

打开网页后，用户应该直接进入应用工作区：

```text
未登录 -> 登录界面
已登录 -> 问题库首页
```

Web 端核心体验：

```text
管理问题库
围绕问题管理回答文章
编辑富文本正文
记录衍生问题
写问题见解和文章见解
绘制流程图
从衍生问题跳转到见解文或首选回答
通过顶部路径回到原阅读上下文
```

## 2. 前端技术边界

当前项目采用：

```text
Next.js
React
TypeScript
Netlify 静态部署
Supabase client
Tiptap 富文本编辑器
@xyflow/react 流程图编辑器
```

由于第一版采用 Netlify 静态部署，数据库里的问题和文章属于运行时动态数据，不适合依赖构建时生成动态页面。

因此第一版路由建议采用：

```text
静态根页面 + 客户端路由状态
```

推荐 URL 形式：

```text
/#/questions
/#/questions/{questionId}
/#/documents/{documentId}
```

好处：

```text
适合静态部署
刷新页面不会要求服务器存在动态路由文件
可以保存和分享当前 Web 状态
后续迁移到真正的 App 路由也容易
```

## 3. 视觉风格

整体风格：

```text
清爽
克制
安静
工具型
高信息密度
适合长期阅读和回看
```

不采用：

```text
营销式首页
大面积渐变
大量装饰图
过大的卡片堆叠
过低的信息密度
```

建议视觉方向：

```text
背景：浅灰白
内容区：白色或极浅灰
主文字：深灰黑
辅助文字：中性灰
边框：浅灰
强调色：蓝绿色或靛蓝
有正文状态：绿色
仅标题状态：黄色或灰黄
无回答状态：灰色
危险操作：红色
```

页面以列表、表格感布局、紧凑工具栏和编辑器为主。

## 4. 页面信息架构

第一版前端页面分为五类：

```text
登录页
问题库首页
回答标题页
正文编辑页
衍生跳转页
```

其中“衍生跳转页”本质上仍然是正文编辑页，只是进入模式不同：

```text
普通正文模式：可以显示衍生问题栏
衍生跳转模式：不再显示衍生问题栏，避免无限跳转
```

## 5. 应用整体布局

桌面端采用工作台布局：

```text
顶部导航栏
主体内容区
右侧面板 / 侧边工具栏
弹窗 / 抽屉 / 浮动面板
```

### 5.1 顶部导航栏

顶部导航栏始终可见。

包含：

```text
应用名称
问题库按钮
回答按钮
当前位置路径
保存状态
用户入口
```

示例：

```text
note-remeber | 问题库 | 回答 | 首页 > 标题页 > 正文页 | 已保存 | 用户
```

规则：

```text
问题库：回到全局问题首页，并清空路径
回答：回到当前原始问题的回答标题页
路径项：每一项都可以点击回到对应位置
保存状态：显示保存中、已保存、保存失败
```

首页问题库页面不显示复杂路径。

### 5.2 主体内容区

主体内容根据当前页面变化：

```text
问题库首页：问题列表
回答标题页：文章标题列表 + 问题见解入口
正文编辑页：富文本编辑器
衍生跳转页：目标正文编辑器或只读查看模式
```

### 5.3 侧边工具栏

正文类页面需要侧边工具栏。

不同正文类型显示不同工具：

```text
文章原文正文：衍生问题、文章见解、流程图
文章见解文：流程图
问题见解文：衍生问题、跳转首选文章、流程图
衍生跳转模式：跳转首选文章 / 跳转见解文、流程图，默认不显示衍生问题
```

工具栏使用图标按钮，并配 tooltip。

## 6. 登录页设计

未登录时显示登录界面。

第一版建议支持：

```text
邮箱验证码登录
```

原因：

```text
不用记密码
适合个人应用
接入 Supabase Auth 简单
```

页面内容：

```text
应用名称
邮箱输入框
发送登录链接按钮
登录状态提示
```

登录成功后：

```text
进入问题库首页
```

登录页不需要营销文案，保持简洁。

## 7. 问题库首页设计

问题库首页是全局问题入口。

### 7.1 页面结构

```text
顶部操作区：
搜索框
新建问题按钮

筛选区：
全部
无回答
有标题
有正文
最近更新

主体：
问题列表
```

### 7.2 问题列表行

问题采用紧凑列表行，不使用大卡片。

每行显示：

```text
问题标题
回答数量
非空正文数量
更新时间
操作按钮
```

示例：

```text
社会运转的规律是什么？        回答 3    正文 1    刚刚更新
什么是自组织？                回答 2    正文 1    昨天
涌现是什么意思？              回答 0    正文 0    4 天前
```

状态表达：

```text
回答 0 · 正文 0：灰色
回答 > 0 · 正文 0：黄色或灰黄
正文 > 0：绿色
```

### 7.3 问题操作

支持：

```text
点击问题进入回答标题页
行内编辑问题标题
删除问题
搜索问题
新建问题
```

删除问题必须二次确认。

### 7.4 新建问题交互

点击新建问题后，出现输入框或小弹窗。

输入过程中实时模糊查询已有问题。

示例：

```text
输入：发动机

下方提示：
发动机对人类的意义是什么
发动机的结构是怎么样的
发动机的原理是什么
```

规则：

```text
完全重名：禁止创建，提示已存在
相似问题：显示提醒，允许选择已有问题或继续新建
```

## 8. 回答标题页设计

回答标题页展示某个问题下的回答文章。

### 8.1 页面结构

```text
问题标题区
问题统计区
问题见解入口
搜索和新建回答标题
回答文章列表
```

示例：

```text
社会运转的规律是什么？
回答 3 · 正文 1

[问题见解]

搜索回答标题...
+ 新建回答

★ 社会系统如何自组织        有正文    有见解    流程图 2
  制度为什么会稳定存在      空正文    无见解    流程图 0
  权力结构的形成机制        空正文    无见解    流程图 0
```

### 8.2 问题标题区

问题标题支持行内编辑。

编辑后需要：

```text
自动保存
检查完全重名
更新顶部路径
```

### 8.3 问题见解入口

需要有明显入口：

```text
打开问题见解
```

显示状态：

```text
未写
已写
内容较少
```

点击后进入问题见解文。

### 8.4 回答文章列表

每行显示：

```text
首选标记
文章标题
正文状态
文章见解状态
流程图数量
衍生问题数量
更新时间
操作按钮
```

操作：

```text
点击标题进入文章原文正文
行内编辑标题
设置为首选回答
删除文章
打开来源链接
```

### 8.5 新建回答标题

新建标题时实时模糊查询当前问题下已有标题。

规则：

```text
同问题下完全重名：禁止创建
相似标题：提示相似项，允许继续创建
```

创建成功后：

```text
立即出现在列表中
默认正文为空
可点击进入正文页
```

## 9. 正文编辑页设计

正文编辑页是最重要的页面。

正文类型：

```text
文章原文正文
文章见解文
问题见解文
```

### 9.1 通用布局

桌面端：

```text
顶部导航栏
正文标题区
富文本工具栏
正文编辑区
右侧内容面板
侧边工具栏
```

结构示意：

```text
顶部：问题库 | 回答 | 首页 > 标题页 > 正文页 | 已保存

正文标题
来源链接 / 状态 / 操作

富文本工具栏

主编辑区                                      右侧面板
[文章正文或见解文]                            [衍生问题 / 流程图 / 见解入口]
```

### 9.2 文章原文正文

文章原文正文显示：

```text
文章标题
来源链接
富文本编辑器
衍生问题侧边栏
文章见解入口
流程图入口
```

主要操作：

```text
编辑文章标题
编辑来源链接
粘贴文章
插入图片
插入表格
记录衍生问题
打开文章见解
打开流程图
删除文章
```

### 9.3 文章见解文

文章见解文显示：

```text
对应文章标题
富文本编辑器
流程图入口
返回文章原文按钮
```

不显示：

```text
衍生问题栏
```

用途：

```text
文章结构梳理
内容总结
个人评价
补充理解
```

### 9.4 问题见解文

问题见解文显示：

```text
问题标题
富文本编辑器
衍生问题侧边栏
跳转首选回答按钮
流程图入口
```

用途：

```text
对整个问题的阶段性回答
综合多个文章后的总结
个人判断和框架
```

如果内容为空：

```text
显示轻提示：还没有写问题见解
```

## 10. 富文本编辑器设计

编辑器采用 Tiptap。

### 10.1 固定工具栏

工具栏包含：

```text
撤销
重做
标题级别
字体
字号
加粗
斜体
下划线
删除线
文字颜色
背景高亮
左对齐
居中
右对齐
有序列表
无序列表
引用
链接
图片
表格
分割线
清除格式
```

按钮使用图标优先，配 tooltip。

### 10.2 浮动工具栏

选中文字时出现浮动工具栏。

包含：

```text
加粗
斜体
下划线
高亮
链接
新建衍生问题
```

“新建衍生问题”可以根据选中文本预填问题标题，例如：

```text
什么是“自组织”？
```

### 10.3 粘贴策略

默认粘贴模式：

```text
保留原格式
```

需要尽量保留：

```text
标题
段落
加粗
斜体
下划线
列表
链接
图片
表格
基础字号和颜色
```

后续可以提供：

```text
保留原格式
清理格式
纯文本粘贴
```

### 10.4 图片策略

图片操作：

```text
粘贴图片
拖拽图片
本地上传图片
从网页复制图片
```

第一版目标：

```text
本地图片和粘贴图片上传到 Supabase Storage
正文中插入图片节点
图片显示加载状态
上传失败可以重试
```

### 10.5 自动保存

正文编辑后自动保存。

建议：

```text
输入停止 800ms - 1500ms 后保存
保存中显示状态
保存成功显示已保存
保存失败显示重试按钮
切换页面前保存未提交内容
```

## 11. 衍生问题侧边栏设计

衍生问题栏只在以下正文中显示：

```text
文章原文正文
问题见解文
```

不显示在：

```text
文章见解文
衍生跳转模式下的目标页面
```

### 11.1 侧边栏结构

```text
标题：衍生问题
搜索框
新建问题输入框
问题列表
```

每个衍生问题显示：

```text
问题标题
回答数量
非空正文数量
是否有问题见解
操作菜单
```

示例：

```text
什么是自组织？
回答 2 · 正文 1 · 有见解

涌现是什么意思？
回答 0 · 正文 0 · 无见解
```

### 11.2 搜索

搜索范围：

```text
当前正文的衍生问题
```

采用字查询。

### 11.3 新建衍生问题

输入时实时查询全局问题库。

规则：

```text
当前正文已有：提示已存在
问题库已有：关联已有问题
问题库没有：创建新问题并关联
```

关联已有问题后，立即刷新显示：

```text
回答数量
非空正文数量
见解状态
```

### 11.4 点击衍生问题

点击衍生问题后弹出跳转选择：

```text
查看问题见解
查看首选回答
```

如果选择的问题没有目标内容：

```text
问题见解为空：允许进入，并提示暂无内容
首选回答无法跳转：提示无法跳转
```

## 12. 衍生跳转页设计

衍生跳转页用于从当前正文临时进入另一个问题的内容。

### 12.1 顶部路径

示例：

```text
首页 > 标题页 > 正文页 > 衍生跳转（见解）
```

继续切换到首选文章：

```text
首页 > 标题页 > 正文页 > 衍生跳转（见解） > 衍生跳转（首选文）
```

每一项都可以点击返回。

### 12.2 返回原正文

衍生跳转页面顶部需要有明确返回入口：

```text
返回原正文
```

返回时尽量恢复原正文滚动位置。

### 12.3 跳转深度限制

在衍生跳转页面中：

```text
不显示衍生问题栏
不允许继续从衍生问题跳到第三个问题
```

允许在同一目标问题内切换：

```text
问题见解文 <-> 首选回答文章
```

这个切换不算第二次衍生跳转。

## 14. 删除交互设计

所有删除必须确认。

删除问题：

```text
删除后会清除该问题下的回答标题、正文、见解文、流程图等内容。确认删除吗？
取消 / 确认
```

删除回答标题：

```text
删除后会清除该标题对应的正文、见解文、流程图和衍生问题引用。确认删除吗？
取消 / 确认
```

删除正文里的衍生问题：

```text
确认从当前正文移除这个衍生问题吗？
取消 / 确认
```

删除流程图：

```text
确认删除这个流程图吗？
取消 / 确认
```

确认按钮使用危险色。

删除成功后显示轻提示。

## 16. 空状态和错误状态

### 16.1 问题库为空

显示：

```text
还没有问题
+ 新建问题
```

### 16.2 问题下没有回答

显示：

```text
这个问题还没有回答文章
+ 新建回答
打开问题见解
```

### 16.3 正文为空

文章原文正文：

```text
粘贴文章内容，或开始输入
```

文章见解文：

```text
写下你对这篇文章的总结、结构和评价
```

问题见解文：

```text
写下你对这个问题的阶段性理解
```

### 16.4 无法跳转

提示：


```text

```

显示：

```text

```

提供重试按钮。

## 17. 响应式设计

第一版主要服务桌面 Web，但需要兼容较窄屏幕。

### 17.1 桌面端

宽度大于 1024px：

```text
顶部导航栏固定
主体内容居中或全宽工作台
正文页显示右侧栏
流程图面板可侧边展开
```

### 17.2 平板和窄屏

宽度 768px - 1024px：

```text
问题列表保持列表
正文右侧栏变成可折叠面板
流程图面板默认覆盖右侧较大区域
```

### 17.3 手机宽度

宽度小于 768px：

```text
顶部路径压缩
问题库和回答页保持单列
正文工具栏可横向滚动
衍生问题变成底部抽屉
流程图默认全屏
```

Web 第一版不需要做到 App 级移动体验，但不能出现文字溢出和功能不可用。

## 18. 可访问性和键盘体验

需要支持：

```text
按钮有可读 tooltip 或 aria-label
弹窗可以 Esc 关闭
确认弹窗有焦点管理
搜索框可键盘选择候选项
新建问题输入框回车创建
富文本工具按钮可键盘聚焦
危险操作不能误触
```

## 19. 前端实现优先级

建议实现顺序：

```text
1. 登录状态和基础 App Shell
2. 问题库首页
3. 新建问题、搜索、去重提示
4. 回答标题页
5. 新建回答标题、设置首选回答
6. 正文编辑页基础布局
7. Tiptap 富文本编辑器
8. 正文自动保存
9. 衍生问题侧边栏
10. 衍生问题跳转选择
11. 问题见解文和文章见解文
12. 顶部路径和返回原正文
13. 流程图面板
14. 删除确认和级联删除入口
15. 响应式细节和空状态
```

## 20. 前端验收清单

第一版前端完成时，需要满足：

```text
未登录时不能进入应用数据页
登录后进入问题库
问题库可以搜索、新建、编辑、删除
新建问题时能显示相似问题
完全重名问题不能创建
回答标题页可以搜索、新建、编辑、删除文章标题
同问题下完全重名标题不能创建
可以设置首选回答
可以打开文章原文正文
正文支持富文本编辑
正文可以自动保存
正文可以显示保存状态
文章原文正文可以打开衍生问题栏
文章原文正文可以打开文章见解
问题见解文可以打开衍生问题栏
文章见解文不显示衍生问题栏
所有可编辑正文都能打开流程图面板
衍生问题显示回答数和正文数
点击衍生问题可以选择跳转目标
无法跳转时有明确提示
衍生跳转后不再显示衍生问题栏
顶部路径可以回到历史节点
问题库按钮可以回到首页
回答按钮可以回到原始问题标题页
所有删除都有确认弹窗
窄屏下主要功能仍然可用
```

## 21. 当前前端方案总结

第一版 Web 前端可以概括为：

```text
一个静态部署的客户端工作台
用问题库作为入口
用回答标题页组织材料
用富文本编辑器保存原文和见解
用衍生问题栏连接问题网络
用顶部路径保证跳转不迷路
用流程图面板辅助理解结构
用轻量但明确的状态提示保护长期使用体验
```
