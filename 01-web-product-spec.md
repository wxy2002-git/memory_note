# Web 端产品功能与逻辑规范 v0.1

本文档用于记录 `note-remeber` 项目的 Web 端产品功能、页面结构和交互逻辑。

当前阶段只讨论产品设计，不包含前端技术、后端技术、数据库结构、接口设计等实现细节。后续技术设计会拆分为新的 Markdown 文档继续补充。

## 1. 产品定位

这是一个以“问题”为中心的阅读记忆应用。

它不是普通的文章收藏夹，也不是传统笔记本，而是帮助用户围绕问题阅读文章、保存回答材料、记录阅读中产生的新疑问、沉淀自己的见解，并在问题网络中快速跳转和回忆。

核心链路：

```text
问题 -> 回答文章 -> 正文阅读 -> 衍生问题 -> 问题见解 / 首选回答 -> 返回原阅读路径
```

应用保存的不只是文章，而是：

```text
问题
回答材料
自己的理解
阅读中产生的新问题
重新找回理解的路径
```

## 2. 核心内容对象

Web 端第一版主要包含以下内容对象：

```text
问题
回答文章标题
文章原文正文
问题见解文
文章见解文
衍生问题
流程图
```

一个问题可以包含：

```text
问题 A
- 问题见解文 A+
- 回答文章 A1
  - 原文正文 A1
  - 文章见解文 A1+
  - 流程图
  - 衍生问题
- 回答文章 A2
- 回答文章 A3
```

衍生问题本质上仍然是全局问题库里的问题，只是它被某篇正文引用了。

例如用户阅读 `A1` 文章时产生了 `问题 B`，那么 `问题 B` 会同时：

```text
出现在 A1 正文的衍生问题栏
出现在全局问题库
可以继续拥有自己的回答文章和问题见解文
```

## 3. 页面结构

Web 端第一版主要包含三个页面层级：

```text
问题库首页
回答标题页
正文编辑页
```

### 3.1 问题库首页

问题库首页是全局入口，显示用户记录过的所有问题。

每个问题需要显示：

```text
问题标题
回答数量
非空正文数量
```

示例：

```text
社会运转的规律是什么？
回答 3 · 正文 1
```

含义：

```text
回答 3：该问题下有 3 个回答文章标题
正文 1：其中有 1 篇文章正文非空
```

问题库首页支持：

```text
新建问题
编辑问题标题
搜索问题
删除问题
查看每个问题的回答数和正文数
区分无回答、有标题、有正文等状态
```

### 3.2 回答标题页

点击某个问题后，进入该问题的回答标题页。

回答标题页显示当前问题下的所有回答文章标题。

示例：

```text
社会运转的规律是什么？

回答 3 · 正文 1

★ A1：社会系统如何自组织        有正文
  A2：制度为什么会稳定存在      空正文
  A3：权力结构的形成机制        空正文
```

每篇回答文章需要显示：

```text
文章标题
是否有正文
是否为首选回答
是否有文章见解
是否有流程图
```

回答标题页支持：

```text
新建回答标题
编辑回答标题
搜索回答标题
删除回答标题
设置首选回答
进入文章原文正文
进入问题见解文
```

每个问题最多只能有一个首选回答。

首选回答用于衍生问题跳转。当其他正文中出现该问题时，用户可以选择跳转到该问题的首选回答文章正文。

### 3.3 正文编辑页

正文编辑页分为三种：

```text
文章原文正文
文章见解文
问题见解文
```

三类正文的功能边界不同。

#### 文章原文正文

文章原文正文用于保存和编辑复制进来的文章内容。

支持：

```text
富文本正文编辑
衍生问题侧边栏
文章见解文入口
流程图入口
```

#### 文章见解文

文章见解文用于记录用户对某篇文章的总结、结构梳理、评价和个人理解。

支持：

```text
富文本见解编辑
流程图入口
```

文章见解文不显示衍生问题栏。

原因是用户对该文章产生的疑问，已经记录在文章原文正文的衍生问题栏里，文章见解文主要承担总结、评价和结构梳理作用。

#### 问题见解文

问题见解文用于记录用户对整个问题的理解、阶段性答案、总结和判断。

支持：

```text
富文本见解编辑
衍生问题侧边栏
流程图入口
跳转到首选回答文章
```

问题见解文可以记录衍生问题，因为用户在总结一个问题时，也可能继续产生新的问题。

## 4. 正文编辑能力

正文编辑器需要接近 Word 式富文本体验，不以纯 Markdown 作为主形态。

正文中需要支持：

```text
加粗
斜体
下划线
删除线
不同字号
不同字体
文字颜色
背景高亮
标题层级
段落
引用
有序列表
无序列表
缩进
链接
图片
表格
分割线
```

复制网页、Word、公众号等内容时，默认尽量保留原格式。

需要尽量保留：

```text
文字样式
标题层级
加粗、斜体、下划线等格式
图片
链接
列表
表格
基础排版结构
```

实际目标不是 100% 复刻原网页，而是尽量保留文章阅读结构和常见格式，避免粘贴后变成纯文本。

图片需要能保存到应用中，避免只依赖外部图片链接。因为外部图片链接未来可能失效。

## 5. 见解文规则

见解文分为两类：

```text
问题见解文
文章见解文
```

### 5.1 问题见解文

每个问题可以有一篇问题见解文。

示例：

```text
问题 A -> A+ 问题见解文
```

它用于记录：

```text
用户对整个问题的理解
阶段性答案
综合总结
个人判断
多个回答文章之间的比较
```

### 5.2 文章见解文

每篇回答文章可以有一篇文章见解文。

示例：

```text
文章 A1 -> A1+ 文章见解文
```

它用于记录：

```text
这篇文章的结构梳理
主要内容总结
观点评价
赞同和不赞同的地方
用户自己的补充理解
```

文章原文正文和文章见解文分开保存。

用户可以只看原文，也可以只看自己的见解。

## 6. 衍生问题规则

可以记录衍生问题的正文类型：

```text
文章原文正文
问题见解文
```

不记录衍生问题的正文类型：

```text
文章见解文
```

衍生问题栏需要显示：

```text
问题标题
回答数量
非空正文数量
```

示例：

```text
什么是自组织？
回答 2 · 正文 1
```

衍生问题栏支持：

```text
新建衍生问题
搜索衍生问题
删除当前正文里的衍生问题引用
点击衍生问题进行跳转
```

删除正文里的衍生问题，只是删除当前正文与该问题的关联。

不会删除问题库里的问题，也不会影响其他正文里引用的同一个衍生问题。

## 7. 衍生问题跳转规则

点击衍生问题时，用户可以选择跳转目标：

```text
跳转到问题见解文
跳转到首选回答文章
```

### 7.1 跳转到问题见解文

如果问题见解文有内容：

```text
进入问题见解文
```

如果问题见解文内容很少或为空：

```text
仍然可以进入
提示内容较少或暂无内容
提供跳转到首选回答文章的按钮
```

问题见解文属于用户自己的理解区，即使为空，也允许进入补写。

### 7.2 跳转到首选回答文章

跳转判断顺序：

```text
有首选回答且正文非空 -> 跳转首选回答正文
没有首选回答但有其他非空正文 -> 跳转第一个有正文的回答
只有标题但正文为空 -> 提示无法跳转
没有回答文章 -> 提示无法跳转
```

提示示例：

```text
该问题已有回答标题，但还没有正文内容，无法跳转。
```

或：

```text
该问题还没有回答，无法跳转。
```

### 7.3 问题见解文内跳转

问题见解文中需要有按钮：

```text
跳转到首选回答文章
```

点击后按同样规则判断：

```text
有首选回答且正文非空 -> 跳转首选回答正文
没有首选回答但有其他非空正文 -> 跳转第一个有正文的回答
没有可读正文 -> 提示无法跳转
```

## 8. 跳转深度限制

衍生问题跳转只能进行一次。

示例：

```text
A1 正文 -> 点击衍生问题 B -> B+ 见解文
```

进入 `B+` 后，不再显示 `B+` 自己的衍生问题栏。

这样做是为了避免无限跳转，防止用户在问题网络里迷路。

但允许在同一个衍生目标内切换：

```text
A1 正文 -> B+ 见解文 -> B1 首选回答正文
```

这个属于同一轮衍生跳转内的“见解文 / 首选文”切换，不算第二次衍生跳转。

进入衍生跳转后的目标页面时，原则如下：

```text
不再继续展示衍生问题侧边栏
保留返回原阅读路径的能力
允许在该目标问题的见解文和首选回答之间切换
```

## 9. 顶部导航路径

应用顶部需要显示当前位置路径。

普通路径示例：

```text
首页 -> 标题页 -> 正文页
```

发生衍生跳转后：

```text
首页 -> 标题页 -> 正文页 -> 衍生跳转（见解）
```

如果继续在同一个衍生目标内跳到首选文章：

```text
首页 -> 标题页 -> 正文页 -> 衍生跳转（见解） -> 衍生跳转（首选文）
```

路径里的每一项都可以点击返回。

顶部还需要固定入口：

```text
问题库
回答
```

规则：

```text
问题库：返回首页，清空路径
回答：返回当前原始问题的回答标题页
```

## 10. 流程图功能

所有可编辑正文都可以创建流程图。

包括：

```text
文章原文正文
文章见解文
问题见解文
```

流程图是挂在具体正文内容上的，不同正文的流程图互相独立。

示例：

```text
问题 A
- 问题见解文 A+
  - 流程图 A+-1
  - 流程图 A+-2

回答文章 A1
- 原文正文 A1
  - 流程图 A1-1
  - 流程图 A1-2
- 文章见解文 A1+
  - 流程图 A1+-1
```

流程图入口放在正文页侧边工具栏。

流程图面板支持：

```text
打开流程图面板
新建流程图
编辑节点
编辑连线
编辑文字
拉伸面板大小
缩小和放大
全屏查看和编辑
保存到当前正文
```

流程图用途：

```text
边读文章边整理文章结构
梳理论证链条
整理因果关系
总结问题框架
辅助理解和复盘
```

## 11. 搜索规则

所有搜索都采用字查询 / 模糊查询。

适用范围：

```text
问题库搜索
回答标题搜索
正文侧边栏衍生问题搜索
新建问题时实时搜索
新建衍生问题时实时搜索
新建标题时实时搜索
```

示例：

已有问题：

```text
发动机的原理是什么
最早的发动机雏形是怎么样的
发动机的结构是怎么样的
```

输入：

```text
发
```

会返回包含“发”的问题。

输入：

```text
发动机
```

会返回所有包含“发动机”的问题。

输入：

```text
发动机的原理
```

结果继续收窄。

## 12. 新建与防重名规则

### 12.1 问题库新建问题

新建问题时，需要实时模糊查询已有问题。

输入过程中，在命名框下方显示相似问题列表。

如果完全重名：

```text
禁止创建
提示已存在该问题
```

如果只是相似：

```text
显示相似问题
用户可以选择已有问题
用户也可以继续新建
```

这样可以减少语义相近但表达略不同的问题重复出现。

### 12.2 正文中新建衍生问题

正文中新建衍生问题时，也需要实时模糊查询全局问题库。

规则：

```text
当前正文已有该衍生问题 -> 禁止重复添加
问题库已有该问题 -> 关联已有问题，不重复创建全局问题
问题库没有该问题 -> 创建新问题，并关联到当前正文
```

关联已有问题后，需要立刻显示它已有的状态：

```text
回答 3 · 正文 2
```

### 12.3 回答标题新建

在某个问题下新建回答标题时，也需要实时模糊查询当前问题下已有标题。

规则：

```text
同一个问题下完全重名 -> 禁止创建
相似标题 -> 显示相似标题
用户可以选择已有标题
用户也可以继续新建
```

## 13. 删除规则

所有删除操作都必须弹出确认框。

基础确认文案：

```text
确认删除吗？
取消 / 确认
```

对于会删除大量关联内容的操作，需要展示更明确的影响范围。

### 13.1 删除问题

删除问题时，需要删除：

```text
问题本身
问题见解文
该问题下所有回答标题
这些标题对应的原文正文
这些标题对应的文章见解文
相关流程图
这些正文里的衍生问题引用
```

但不删除这些正文中引用过的衍生问题本身。

如果用户想删除某个衍生问题本身，需要在问题库中单独删除该问题。

建议确认文案：

```text
删除后会清除该问题下的回答标题、正文、见解文、流程图等内容。确认删除吗？
```

### 13.2 删除回答标题

删除回答标题时，需要删除：

```text
回答标题
对应原文正文
文章见解文
相关流程图
该正文里的衍生问题引用
```

但不删除这些衍生问题在问题库中的本体，也不影响其他正文对这些问题的引用。

建议确认文案：

```text
删除后会清除该标题对应的正文、见解文、流程图和衍生问题引用。确认删除吗？
```

### 13.3 删除正文里的衍生问题

删除正文里的衍生问题时：

```text
只删除当前正文与该问题的关联
不删除问题库里的问题
不影响其他正文
```

建议确认文案：

```text
确认从当前正文移除这个衍生问题吗？
```

## 14. Web 端第一版核心目标

第一版需要优先保证以下能力顺畅：

```text
问题能创建、搜索、去重、删除
问题下能创建回答标题
回答标题能搜索、去重、删除
文章正文能富文本保存
复制内容时能尽量保留原格式
文章正文能记录衍生问题
衍生问题能显示回答数和正文数
能写文章见解
能写问题见解
能设置首选回答
能从衍生问题跳转到问题见解文
能从衍生问题跳转到首选回答文章
能从跳转页面返回原阅读路径
跳转深度受限制，避免无限跳转
所有可编辑正文都能创建流程图
所有删除操作都有确认
```

## 15. 当前设计总结

这个应用的核心体验是：

```text
围绕问题收集文章
阅读文章时记录衍生问题
给文章写自己的总结
给问题写自己的阶段性答案
用流程图辅助理解结构
从衍生问题跳到自己的见解或首选原文
跳完后还能回到原来的阅读路径
```

一句话总结：

```text
note-remeber 要保存的是问题、回答材料、个人理解、阅读疑问，以及重新找回理解的路径。
```
